# 前端选择Node.js来做后端开发
1. 小公司 掌握一门后端语言,无论独立开发 还是前端团队做部分后台服务的时候，主动承担部分工作,凸显个人优势
2. 大公司 有利于和后端 运维同事沟通。有利于提升自己编码能力。
# 是什么阻碍了前端对于 Node.js 的进阶？
1. 核心技术点的掌握不足  官网的技术文档不能满足Nodejs应用开发的技术能力要求
2. 关注点及流程 方法的不同
    - 后台服务的核心就是能够安全 稳定地提供服务,避免出现服务异常导致无法提供服务的问题。
    - 性能分析和优化都是非常重要的环节。
    - 服务端则基于数据和数据流转。 UML类图 时许图

# 核心知识点 + 技术应用分析 + 研发实战
1. 核心知识点  Node.js原理 RESTful服务实践 当前流行架构的对比  多进程方案  性能影响因素 CPU过载保护方案  密集I/O设计思想 以及缓存的介绍和应用。  
2. 技术应用分析  系统稳定  内存安全 网络安全 性能分析的解决方法。  开源工具实现服务上线前自动化的系统安全检查。
3.   RPC服务实践  通用抢票系统  通用投票系统以及当前最为流行的Serverless.




* 工程化侧重于开发者提升 研发效率或者研发质量
* 后端服务 稳定安全 并发性能  扩容 性能优化


* 前端工程化的应用着重于开发效率的提升和研发效率的保证。
* 后端服务 发挥Node.js的异步驱动特性 在主线程不被CPUT密集型所影响时,发挥Node.js高并发特性,作为大部分网络I/O较高的后端服务。
